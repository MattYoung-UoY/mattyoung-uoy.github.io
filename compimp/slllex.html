Straight Line Language Lexer


Setup:


I downloaded the skeleton code from the Author's website, and put it into an Eclipse environment.

Creating the Lexer:


To generate the Lexer using JFlex, I needed to create a specification file that would tell JFlex what functionality I would like in the generated Lexer.
To begin with, I specified the package that the Lexer class would be contained within, and which classes it would need to import:

package lex;

import java_cup.runtime.Symbol;
import errorMsg.ErrorMsg;
import java.io.BufferedReader;

After this I began to specify required features that my desired Lexer would require:

%class Yylex
%unicode
%line
%column
%cup
%char

%class Yylex specifies that the name of the resulting class file should be Yylex.
%unicode specifies that the source files will use unicode encoding.
%line and %comlumn enable counters to keep track of the current line and column that the Lexer is reading. This allows error messages to provide useful information about where the error occured.
%cup enables "Cup compatibility mode" which defines the method that returns the next token to be called next_token which returns an object of java_cup.runtime.Symbol. This also specifies what to do when the Lexer reads the End Of File (EOF) marker, however we overwrite this later. It also causes the Lexer to close the input stream after reading the file.
%char enables a counter to count how many characters have been read since the start of the file. Like %line and %column, this is also useful for user-friendly error reporting.
